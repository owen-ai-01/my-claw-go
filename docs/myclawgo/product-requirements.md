# MyClawGo 产品需求文档（PRD）

> 版本：v1.0
> 更新时间：2026-04-04
> 定位：面向非技术用户的 AI 办公室 SaaS 平台

---

## 一、产品定义

### 一句话定位
**MyClawGo = 每个人都能拥有的私人 AI 员工团队，注册即用，零技术门槛。**

### 完整定义
MyClawGo 是一个托管式 AI Agent 平台。用户注册后，系统自动为其创建一个独立隔离的 AI 工作空间（内含完整的 OpenClaw runtime），用户无需配置服务器、无需管理 API Key，即可在网页端：

- 创建和配置多个 AI 员工（Agent）
- 与 Agent 对话、分配任务
- 组建 AI 团队进行多 Agent 协作
- 通过 Telegram 随时随地与 Agent 沟通
- 设置定时任务让 Agent 自动工作
- 在"办公室"视角实时查看所有 Agent 的工作状态

---

## 二、目标用户

### 核心用户画像
| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|
| 独立创业者 | 一个人跑多条业务线，需要 AI 分担日常工作 | 省时、自动化、随时响应 |
| 内容创作者 | 需要 AI 辅助写作、调研、发布内容 | 创意协作、知识沉淀、多渠道触达 |
| 小团队负责人 | 团队 5-20 人，希望用 AI 提升运营效率 | 任务自动化、状态可视、团队协作 |
| 中文用户 | 主要使用中文工作，需要高质量中文 AI 响应 | 中文优化模型、本地化体验 |

### 核心反诉求（用户不想要的）
- 不想配置服务器和环境
- 不想管理 OpenAI / Anthropic API Key
- 不想看懂 JSON 配置文件
- 不想因 AI 卡住而无从知晓
- 不想在多个 AI 工具间反复切换

---

## 三、产品价值主张

1. **私有隔离**：每个用户独立容器，数据不共享，Agent 记忆持久化
2. **零技术门槛**：注册即用，所有配置通过网页完成
3. **AI 团队感**：多 Agent 协作，有分工、有状态、有执行记录
4. **随时随地**：Telegram 接入，手机上也能管理和对话 Agent
5. **自动化能力**：定时任务让 Agent 持续工作，不需要人盯着
6. **成本透明**：按 Credit 计量，订阅制 + 按量购买灵活选择

## 四、功能模块需求

### 4.1 账号与认证

**注册与登录**
- 支持邮箱 + 密码注册
- 支持 Google / GitHub 一键登录
- 注册后自动触发工作空间初始化（容器创建）
- 邮件验证机制

**用户资料**
- 修改显示名称、上传头像

---

### 4.2 Agent 管理（核心功能）

#### 4.2.1 Agent 列表（Office / Settings → Agents）
- 展示当前工作空间所有 Agent
- 每个 Agent 卡片显示：头像、名称、角色、当前状态（在线/忙碌/空闲/离线）、最近活动时间
- 支持新建 Agent、删除 Agent

#### 4.2.2 新建 Agent
表单字段：
- Agent ID（唯一标识，英文字母+数字+连字符，创建后不可修改）
- 显示名称（中英文均可）
- Emoji（用于快速识别）
- 角色 / 岗位（如：内容编辑、数据分析师、客服代表）
- 所属部门（可选，用于分组显示）
- 默认模型（从可用模型列表选择）

#### 4.2.3 Agent 配置（每个 Agent 的详情页）
**基础信息 Tab**
- 修改名称、Emoji、角色、部门、头像
- 默认模型切换

**知识配置 Tabs（Markdown 编辑器）**

| Tab 名称 | 用途 | 典型内容 |
|---------|------|---------|
| IDENTITY.md | Agent 的人格与行为风格 | "你是 Sarah，一位有 10 年经验的内容策略师..." |
| USER.md | 用户自身信息，让 Agent 了解你 | 用户的目标、背景、偏好、约束 |
| SOUL.md | Agent 的核心价值观与决策框架 | 优先级排序、边界原则 |
| TOOLS.md | Agent 可使用的工具与集成能力 | 可调用 API、插件说明 |
| AGENTS.md | 整个工作空间的 Agent 名册 | 全员列表，支持 Agent 互相了解 |

**Telegram 配置 Tab**
- 填写 Bot Token（从 Telegram BotFather 获取）
- 系统自动生成 Webhook URL
- 一键绑定/解绑
- 显示绑定的 Telegram 聊天（Chat ID、群/私聊）

---

### 4.3 对话（Chat）

#### 4.3.1 基础对话
- 左侧 Agent 列表，点击切换当前对话 Agent
- 支持 Markdown 渲染
- 消息带时间戳
- 流式响应（打字机效果）
- 历史消息分页加载

#### 4.3.2 群组对话（Group Chat）
- 创建群组，设置群组名称、描述、公告
- 添加多个 Agent 为成员，指定一个 Leader Agent（主负责人）
- 用户发送消息 → Leader 优先响应
- Agent 通过 `@agentId` 主动移交给下一个 Agent（Relay 接龙模式）
- 用户也可以直接 `@agentId` 指定特定 Agent 响应
- 自动接龙保护：最大接龙轮数可配置（1-20 轮），防止无限循环
- 停止指令：输入 `#stop` 或 `#pause` 停止接龙

**群组配置项：**
- 群名称、描述、公告（支持 Markdown）
- 成员列表（可增删 Agent）
- Leader 指定
- Relay 开关及最大轮数
- 接龙冷却时间（ms）

---

### 4.4 办公室（Office）

**核心目标：让用户一眼知道所有 Agent 在干什么。**

#### 页面内容
- Agent 状态卡片网格，每张卡片显示：
  - 头像 + 在线状态指示灯
  - 名称 + 角色
  - 当前状态：`idle / chatting / running / waiting / offline / error`
  - 正在执行的任务摘要（截取前 50 字）
  - 最近活动时间（"3 分钟前"）
  - 消息来源（Chat / Telegram / 定时任务）

- 实时活动日志流：
  - 滚动展示最近所有 Agent 的操作记录
  - 包含：时间、Agent 名称、动作类型、简短摘要

---

### 4.5 定时任务（Tasks）

#### 任务创建
- 选择目标 Agent
- 任务名称
- 调度类型：
  - `定时重复`：每隔 X 小时/分钟
  - `Cron 表达式`：精细控制（如每周一 9 点）
  - `单次执行`：指定时间执行一次
- 任务内容（发给 Agent 的消息文本）
- 可选模型覆盖（临时换用不同模型执行此任务）

#### 任务管理
- 任务列表，显示：名称、调度规则、下次执行时间、启用状态
- 开启/暂停任务
- 立即执行（手动触发）
- 执行历史记录（时间、耗时、状态、消耗 Credit）

---

### 4.6 Telegram 集成

**面向用户的操作流程：**
1. 打开 Agent 设置 → Telegram Tab
2. 从 BotFather 创建 Bot，复制 Bot Token
3. 粘贴到配置框，点击「绑定」
4. 系统自动完成 Webhook 设置
5. 打开 Telegram，找到对应 Bot，开始对话
6. Agent 的回复实时出现在 Telegram

**支持场景：**
- 用户与 Agent 的私聊（1对1）
- 将 Bot 加入 Telegram 群组（群消息路由到对应 Agent）

**消息同步：**
- Telegram 发送的消息也会出现在网页 Chat 的历史记录中
- 双向同步，两端均可查看完整对话历史

| 包名 | Credits | 价格 |
|-----|---------|-----|

**Credit 消耗规则：**
- 每条消息根据使用的模型和 Token 数量扣减 Credit
- 系统智能路由（简单对话用便宜模型，复杂任务用强力模型），自动节省 Credit
- 用户可查看每次对话的 Credit 消耗明细

**Credit 余额不足处理：**
- 余额低时发送邮件提醒
- 余额耗尽时对话被暂停，引导用户购买 Credit 或升级订阅
- 当月订阅 Credit 到期时需续费或购买额外包

## 五、用户核心旅程

### 旅程一：新用户注册并开始使用
```
1. 访问首页 → 点击「注册」
2. 邮箱注册或 Google 一键登录
3. 系统自动为用户初始化 AI 工作空间（后台创建容器）
4. 跳转到 Chat 页面，看到默认 Agent 已就绪
5. 直接开始对话
```

### 旅程二：配置自己的 AI 员工
```
1. 进入 Settings → Agents
2. 点击「新增 Agent」，填写名称、角色、模型
3. 进入 Agent 详情页 → IDENTITY.md Tab
4. 编写 Agent 的人格描述（系统提供模板参考）
5. 保存后，切换到 Chat 页面，与新 Agent 对话
```

### 旅程三：通过 Telegram 与 Agent 沟通
```
1. 进入 Agent 设置 → Telegram Tab
2. 从 BotFather 创建 Bot，获取 Token
3. 粘贴 Token，点击「连接」
4. 打开 Telegram，找到 Bot 并发送「/start」
5. Agent 开始响应，如同在 Web 端对话
```

### 旅程四：组建 AI 协作团队
```
1. 创建多个专职 Agent（如：研究员、文案、审核员）
2. 进入 Groups → 新建群组
3. 将相关 Agent 加入群组，指定 Leader
4. 发送任务消息 → Leader 完成初步工作
5. Leader 通过 @成员 移交给下一个 Agent
6. 多 Agent 接力完成复杂任务
```

### 旅程五：设置自动化定时任务
```
1. 进入 Tasks → 新建任务
2. 选择 Agent（如：数据分析师）
3. 设置调度：每天早 9 点
4. 填写任务内容：「生成昨日数据汇总报告」
5. 任务自动执行，结果保存在 Chat 历史中
```

## 六、非功能性需求

### 可靠性
- 每个用户的工作空间独立隔离，单个用户异常不影响其他用户
- 容器意外重启后，Agent 配置和聊天记录自动恢复
- Telegram Webhook 接收失败自动重试

### 响应速度
- 对话首 Token 响应时间 < 3 秒
- Office 页面加载 < 2 秒
- 历史消息分页加载 < 1 秒

### 数据安全
- 用户间数据完全隔离
- Bot Token 等敏感信息加密存储
- HTTPS 全程加密传输
- Stripe PCI DSS 合规支付

### 国际化
- 支持中文（简体）/ 英文双语界面
- AI 模型支持中文优化路由（DeepSeek-V3 处理中文内容）
