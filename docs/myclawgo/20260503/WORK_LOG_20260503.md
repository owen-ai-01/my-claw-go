## 三、VPS Provisioning 失败 Bug 修复

### 现象
用户付费后出现：
```
❌ Workspace creation failed
We were unable to set up your workspace.
```
VPS 已在 Hetzner 上创建成功，但 DB 状态为 `failed`。

### 根本原因

`register/route.ts` 的 `authSteps` 中，第一条命令是：
```bash
chown -R openclaw:openclaw /home/openclaw/.openclaw/agents
```
但 `agents/` 目录在 gateway **首次运行前不存在**（gateway 在第一次使用时才会创建它）。`chown` 失败后整个 `&&` 链终止，后续写入 OpenRouter key、重启 gateway 均未执行，deploy 抛出异常，DB 被标记为 `failed`。

### 代码修复

`src/app/api/internal/runtime/register/route.ts`：

```diff
- const authSteps = openrouterKey
-   ? `chown -R openclaw:openclaw /home/openclaw/.openclaw/agents && \
-      mkdir -p /home/openclaw/.openclaw/agents/main/agent && \
+ const authSteps = openrouterKey
+   ? `mkdir -p /home/openclaw/.openclaw/agents && \
+      chown -R openclaw:openclaw /home/openclaw/.openclaw/agents && \
+      mkdir -p /home/openclaw/.openclaw/agents/main/agent && \
       ...`
-   : `chown -R openclaw:openclaw /home/openclaw/.openclaw/agents`;
+   : `mkdir -p /home/openclaw/.openclaw/agents && chown -R openclaw:openclaw /home/openclaw/.openclaw/agents`;
```

在 `chown` 前加 `mkdir -p`，确保目录存在。
