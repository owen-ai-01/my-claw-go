## 2) 技术审计（jinma 视角）
> 同上：本轮为本地等价技术审计。

### 风险
- **高风险**：sessionId 可被猜测/泄露后，可能被他人调用 runtime chat/exec。
- **中风险**：无效容器启动导致资源浪费，放大并发负载。

### 实现方案
1. 在 runtime chat/exec 入口前置强鉴权：必须登录且 `currentUserId === sessionId`。
2. 在 runtime start 中先做余额判断，再 `ensureUserContainer`。
